# カスタマージャーニー

Atlas Platformにおけるカスタマージャーニーは、**6段階で構成される設定可能なフロー**です。中核となる流れは全市場で共通ですが、ロイヤルティ、プロモーション、スケジュール注文などの主要な接点は地域チームが市場ごとに有効化・無効化できます。

## 6つのステップ

| ステップ | 名称 | 説明 |
|---|---|---|
| 1 | **検索・発見** | お客様がアプリ、Webサイト、アグリゲーター経由でKFCを見つける |
| 2 | **店舗選択** | 位置情報または配送先住所に基づいて店舗を選ぶ |
| 3 | **メニュー閲覧** | 店舗ごとにローカライズされたメニューを閲覧する |
| 4 | **カート・カスタマイズ** | 商品を追加し、モディファイアやセット内容を選ぶ |
| 5 | **ロイヤルティとプロモーション** | 市場設定に応じてポイント、バウチャー、プロモコードを適用 |
| 6 | **チェックアウト** | 支払い、注文確定、デジタルレシート受領 |

## ステップ5 — ロイヤルティとプロモーション

この工程は市場差が最も大きい部分です。各市場が独自にロイヤルティとプロモーションのルールを管理します。

### Japan 🇯🇵
KFC Japanでは、KFC Japanアプリに統合された**ポイント制ロイヤルティプログラム**を運用しています。お客様は各注文でポイントを獲得し、チェックアウト時に利用できます。プロモコードにも対応しており、多くのキャンペーンではポイントとの併用が可能です。

### Australia 🇦🇺
KFC Australiaでは、KFCアプリ内で提供される**バウチャー方式**を採用しています。お客様は有効なディールを確認し、注文前にバウチャーを選択すると、チェックアウト時に自動適用されます。加えて、キャンペーン専用のプロモコード入力欄もあります。

## 市場別設定マトリクス

| 機能 | Japan 🇯🇵 | Australia 🇦🇺 |
|---|---|---|
| Loyalty points earn & redeem | ✅ | — |
| Voucher selection at ordering | — | ✅ |
| Promo code at checkout | ✅ | ✅ |
| Combo upsell prompt | ✅ | ✅ |
| Scheduled future orders | ✅ | — |
| Guest checkout (no login) | ✅ | ✅ |
| Digital receipt via email | ✅ | ✅ |

## よくある質問

**お客様はロイヤルティ特典とプロモコードを同時に使えますか。**
はい。両方がその市場で有効になっていれば併用できます。先にプロモコードが適用され、その後の割引後金額に対してロイヤルティポイントが計算されます。この挙動は市場ごとに Admin Portal → Market Settings → Loyalty Rules で設定できます。

**チェックアウト中にお客様がオフラインになった場合はどうなりますか。**
アプリはメニューとカートをローカルに保持しますが、チェックアウトには通信が必要です。接続が切れた場合、注文内容はカートに保持され、通信復旧後に再試行を促されます。

**特定の市場でゲストチェックアウトを無効にできますか。**
はい。これは市場レベルのトグルで、Admin Portal の Market Settings → Customer Journey から変更できます。

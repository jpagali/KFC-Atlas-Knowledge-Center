# Release 2026.03.20

Released on **March 20, 2026**.

This release tightens the Byte Commerce system architecture interaction so pinned notes stay readable while users keep exploring the diagram, completes a major Japanese localization pass across the Atlas Wiki, and makes Byte capability technical sources easier to trace back to their FRD references.

## Highlights

- **Pinned architecture notes now stay visible while the diagram scrolls**
  Users can pin a note, continue scrolling through the architecture map underneath it, and keep the note in view for reference.

- **Pinned note content now updates without dropping pinned mode**
  Once a note is pinned, clicking another system box swaps the note content immediately while keeping the panel pinned.

- **Embedded architecture sizing now expands correctly**
  The platform explorer and docs iframe now resize with the architecture panel so pinned notes no longer get clipped inside the embed.

- **Japanese coverage now spans the full docs set**
  Every English documentation page now has a Japanese counterpart, including Byte Capabilities, release notes, and the new daypart guides.

- **The platform explorer now localizes in Japanese**
  The Japanese mental model page now opens a Japanese-aware explorer, including translated explorer chrome, hierarchy content, and system architecture copy.

- **Byte capability FRD references now deep-link into source requirements**
  Technical source blocks across the Byte capability docs now link explicit Helium, approved Portal, and approved Commerce Backend FRD references back to their source requirements pages.

## What Changed

### System Architecture Diagram

- Restored the `Pin this note` interaction so hover and click still preview a box before pinning
- Updated pinned notes to behave like a persistent on-screen overlay while the underlying diagram keeps scrolling
- Kept pinned mode active when switching between boxes so users can compare systems without re-pinning

### Embedded Explorer Sizing

- Added height reporting from the architecture diagram to the platform explorer wrapper
- Added height reporting from the platform explorer back to the docs iframe component
- Updated the mental model page embed to listen for live height changes from the explorer

### Japanese Localisation

- Added Japanese versions of all previously missing docs, including Byte Capabilities domains, enablement guides, references, release notes, and daypart documentation
- Updated Japanese navbar, footer, homepage, What's New, Contribute, and Atlas Peek copy to reduce English fallback text
- Regenerated Japanese search index content so the new translated docs are discoverable

### Platform Explorer Localisation

- Updated the Japanese mental model page to pass locale into the platform explorer embed
- Localized the platform explorer shell so the Japanese page shows Japanese headings, tabs, panel labels, and footer copy
- Localized the embedded system architecture view with Japanese labels, legend text, panel UI, and translated box descriptions for the architecture canvas

### Byte Capability Source Linking

- Linked explicit `FRD-HEL-*` references in Byte capability technical source sections directly to their matching Helium requirements anchors
- Expanded compact Helium FRD ranges into individual linked references where needed so each requirement can be opened directly
- Linked approved `FRD-BCOM-*` and normalized `FRD-PORTAL-*` references where exact mappings were available, while leaving unmapped codes as plain text

## In Plain English

The architecture diagram is much easier to use now, the Japanese experience is much more complete, and the Byte capability docs are more useful as a source map. You can pin a note, keep it on screen while exploring the system map, and jump from technical source references straight into the underlying requirements when those mappings are available.
